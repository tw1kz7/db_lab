## Таблиці
---
- `users` - користувачі
- `developers` - розробники
- `employers` - роботодавці
- `projects` - проєкти

## SQL-скрипт бази даних
---
db.sql
```sql
DROP TABLE IF EXISTS projects;
DROP TABLE IF EXISTS developers;
DROP TABLE IF EXISTS employers;
DROP TABLE IF EXISTS users;

CREATE TABLE users (
	user_id SERIAL PRIMARY KEY,
	user_name VARCHAR(30) NOT NULL,
	user_surname VARCHAR(50) NOT NULL,
	user_country VARCHAR(100) NOT NULL,
	user_email VARCHAR(100) NOT NULL UNIQUE,
	user_phone VARCHAR(18) NOT NULL
);

CREATE TABLE developers (
	dev_id INT PRIMARY KEY REFERENCES users(user_id) ON DELETE CASCADE,
	is_searching_job BOOLEAN NOT NULL DEFAULT TRUE,
	dev_links TEXT
);

CREATE TABLE employers (
	employer_id INT PRIMARY KEY REFERENCES users(user_id) ON DELETE CASCADE,
	employer_company VARCHAR(150) NOT NULL,
	employer_position VARCHAR(100) NOT NULL
);

CREATE TABLE projects (
	project_id SERIAL PRIMARY KEY,
	dev_id INT NOT NULL REFERENCES developers(dev_id) ON DELETE CASCADE,
	project_name VARCHAR(150) NOT NULL,
	project_description TEXT NOT NULL,
	project_url VARCHAR(255) NOT NULL,
	project_category VARCHAR(50) NOT NULL,
	project_active BOOLEAN NOT NULL DEFAULT TRUE
);
```

## Агрегаційні запити та групування

```sql
SELECT user_country, COUNT(*) AS total_users
FROM users
GROUP BY user_country
ORDER BY total_users DESC;
```
Обʼєднує користувачів у групи за країною та підраховує кількість зареєстрованих акаунтів у кожній країні та сортує їх від більшого до меншого

```sql
SELECT dev_id, MIN(project_id) AS first_project_id, MAX(project_id) AS last_project_id
FROM projects
GROUP BY dev_id;
```
Групує записи в таблиці `projects` за ідентифікатором розробника та знаходить номер найпершого та останнього завантаженого проєктів

```sql
SELECT project_category,
       SUM(CASE WHEN project_active = TRUE THEN 1 ELSE 0 END) AS active_projects_sum,
       AVG(CASE WHEN project_active = TRUE THEN 1.0 ELSE 0.0 END) AS active_projects_avg
FROM projects
GROUP BY project_category;
```
У таблиці `projects` рядок `project_active` збирає `TRUE` або `FALSE` . Запит рахує кожен `TRUE` як одиницю і рахує середню кількість активних проєктів серед всіх проєктів що є

```sql
SELECT project_category, COUNT(*) AS projects_count
FROM projects
GROUP BY project_category
HAVING COUNT(*) >= 2;
```
Групує проєкти за їх категоріями, рахує кількість проєктів у кожній категорії, відсікає та не виводить ту категорію у якій кількість проєктів менша за 2 у сумі

## Джоіни

```sql
SELECT u.user_name, u.user_surname, p.project_name, p.project_category
FROM users u
INNER JOIN developers d ON u.user_id = d.dev_id
INNER JOIN projects p ON d.dev_id = p.dev_id;
```
Запит звʼязує таблицю користувачів з типом акаунту "розробник" та проєктами. Якщо у користувача немає проєктів або він не є розробником - `INNER JOIN` не виведе його в результат

```sql
SELECT u.user_name, u.user_surname, e.employer_company, e.employer_position
FROM users u
LEFT JOIN employers e ON u.user_id = e.employer_id;
```
Запит виводить всіх користувачів з таблиці `users`. Якщо користувач це роботодавець то у таблицю підставлять дані про компанію та посаду. Якщо розробник - в цих полях буде значення `NULL`

```sql
SELECT u.user_name, u.user_surname, d.dev_links
FROM projects p
RIGHT JOIN developers d ON p.dev_id = d.dev_id
INNER JOIN users u ON d.dev_id = u.user_id
WHERE p.project_id IS NULL;
```
Бере всіх учасників із правої таблиці `developers` і шукає їх проєкти в таблиці `projects`. Фільтр залишає лише тих розробників у яких немає жодного проєкту

## Запити з використанням підзапитів

```sql
SELECT user_name, user_surname
FROM users
WHERE user_id IN (
    SELECT dev_id 
    FROM projects
);
```
Внутрішній запит просто збирає всі `dev_id` з таблиці `projects` , головний запит дивиться на цей список і виводить лише тих користувачів у яких є хоча б один проєкт

```sql
SELECT user_name,
       (SELECT COUNT(*) FROM projects WHERE dev_id = users.user_id) AS total_projects
FROM users;
```
Головний запит бере користувача з таблиці `users` а внутрішній підзапит рахує кількість його записів у таблиці `projects` 

```sql
SELECT project_category, COUNT(*) AS total_count
FROM projects
GROUP BY project_category
HAVING COUNT(*) > (
    SELECT COUNT(*) 
    FROM employers
);
```
Головний запит групує проєкти за категоріями та рахує їх. Блок `HAVING` порівнює цю кількість із загальним числом роботодавців у базі і залишає найбільші категорії.
