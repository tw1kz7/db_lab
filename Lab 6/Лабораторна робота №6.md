## Перша міграція
---
Нова модель для відгуків
```prisma
model reviews {
  review_id   Int      @id @default(autoincrement())
  rating      Int
  comment     String?
  project_id  Int
  projects    projects @relation(fields: [project_id], references: [project_id], onDelete: Cascade)
}
```

Як це виглядає у `projects`
```prisma
model projects {
  project_id          Int         @id @default(autoincrement())
  dev_id              Int
  category_id         Int?
  project_name        String      @db.VarChar(150)
  project_description String
  project_url         String      @db.VarChar(255)
  project_active      Boolean     @default(true)
  categories          categories? @relation(fields: [category_id], references: [category_id], onUpdate: NoAction)
  developers          developers  @relation(fields: [dev_id], references: [dev_id], onDelete: Cascade, onUpdate: NoAction)
  reviews     reviews[]
}
```

```bash
npx prisma migrate dev --name add-reviews-table
```

## Друга міграція
---
Додавання нового поля `balance` у модель `users`
```prisma
model users {
  user_id      Int       @id @default(autoincrement())
  user_name    String    @db.VarChar(30)
  user_surname String    @db.VarChar(50)
  user_country String    @db.VarChar(100)
  user_email   String    @unique @db.VarChar(100)
  user_phone   String    @db.VarChar(18)
  balance      Float     @default(0.0)

  developers   developers?
  employers    employers?
}
```

Міграція
```bash
npx prisma migrate dev --name add-user-balance
```

## Третя міграція
---
Видалення рядку `user_phone` з `users`

```prisma
model users {
  user_id      Int       @id @default(autoincrement())
  user_name    String    @db.VarChar(30)
  user_surname String    @db.VarChar(50)
  user_country String    @db.VarChar(100)
  user_email   String    @unique @db.VarChar(100)
  balance      Float     @default(0.0)

  developers   developers?
  employers    employers?
}
```

Команда міграції
```bash
npx prisma migrate dev --name drop-user-phone
```
