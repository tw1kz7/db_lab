## **Пояснювальна записка до бази даних**

Використовуючи мою ER-діаграму було створено наступні таблиці:
- `users` - зберігає користувачів платформи
- `developers` - розробники

## **Таблиці**

Розглянемо кожну таблицю:

### `users` 
Ця таблиця містить інформацію про користувачів, ідентифікатор, імʼя, призвіще, країну, електронну пошту та номер телефону
- `user_id` - ідентифікатор
- `user_name` - імʼя користувача
- `user_surname` - призвіще користувача
- `user_country` - країна користувача
- `user_email` - електронна пошта
- `user_phone` - номер телефону

### `developers`
Ця таблиця містить інформацію користувачів, з типом акаунту "розробник"
- `dev_id` - ідентифікатор розробника
- `is_searching_job` - чи шукає розробник роботу, приймає значення TRUE або FALSE
- `dev_links` - посилання на соц мережі

## `employers`
Ця таблиця містить інформацію про користувачів, з типом акаунту "роботодавець"
- `employer_id` - ідентифікатор роботодавця
- `employer_company` - компанія яку представляє роботодавець
- `employer_position` - роль роботодавця

### `projects` 
Ця таблиця містить інформацію про проєкти користувачів
 - `project_id` - ідентифікатор проєкту
 - `dev_id` - ідентифікатор розробника
 - `project_name` - імʼя проєкту
 - `project_description` - опис проєкту
 - `project_url` - посилання на сайт
 - `project_category` - категорія проєкту
 - `project_active` - статус проєкту, може бути прихованим або активним

## **SQL-скрипт**
```sql
DROP TABLE IF EXISTS projects;
DROP TABLE IF EXISTS developers;
DROP TABLE IF EXISTS employers;
DROP TABLE IF EXISTS users;

CREATE TABLE users (
	user_id SERIAL PRIMARY KEY,
	user_name VARCHAR(30) NOT NULL,
	user_surname VARCHAR(50) NOT NULL,
	user_country VARCHAR(100) NOT NULL,
	user_email VARCHAR(100) NOT NULL UNIQUE,
	user_phone VARCHAR(13) NOT NULL
);

CREATE TABLE developers (
	dev_id INT PRIMARY KEY REFERENCES users(user_id) ON DELETE CASCADE,
	is_searching_job BOOLEAN NOT NULL DEFAULT TRUE,
	dev_links TEXT
);

CREATE TABLE employers (
	employer_id INT PRIMARY KEY REFERENCES users(user_id) ON DELETE CASCADE,
	employer_company VARCHAR(150) NOT NULL,
	employer_position VARCHAR(100) NOT NULL
);

CREATE TABLE projects (
	project_id SERIAL PRIMARY KEY,
	dev_id INT NOT NULL REFERENCES developers(dev_id) ON DELETE CASCADE,
	project_name VARCHAR(150) NOT NULL,
	project_description TEXT NOT NULL,
	project_url VARCHAR(255) NOT NULL,
	project_category VARCHAR(50) NOT NULL,
	project_active BOOLEAN NOT NULL DEFAULT TRUE
);
```

`DROP TABLE` - використовується для того, щоб перезапустити таблицю без помилок
`ON DELETE CASCADE` - для того, щоб видаляти інформацію про користувача з усіх таблиць, щоб уникнути помилки
